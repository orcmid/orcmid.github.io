When an ODMA application loads ODMA32.DLL, the Connection Manager will look in the
registry and supply the default DMS for that user.  So, if your DMS is installed 
in the registry as the default then Word should access your code automatically.  
To make sure you are the default DMS, go to the registry, and open the 
HKEY_CLASSES_ROOT\ODMA32 key.  Your DMS id should be installed here.  
Lets say you are using ODMASAMP as your default DMS.  Under that id you install 
the key:  DEFAULT.  Make sure that none of the other DMSs have the default key, 
though.  The Connection Manager will use the first default it finds in the list.

If you are running on an NT machine, you may need to add a registry entry to get 
Word to work properly with ODMA.  Check for the following key:

HKEY_LOCAL_MACHINE\Software\Microsoft\ODMA32\ANSIonNT       (it has no value set)

Hope this helps.
