Summary

This FAQtip describes the basic Workflow Approach.  Links for downloading the ODMA Workflow 1.0 specification and the ODMA Workflow 1.0 software are also included.

1. ODMA Workflow Approach

[tbd]

Acknowledge John Newton's persistence in having a Workflow Companion.

Indicate the parallel-connection-manager approach adopted for ODMA Workflow 1.0

Indicate that software was produced and the SDK is available, but tested only with ODMA 1.0 libraries as far as we know.

Indicate abandonment of Workflow 1.0 support as part of ODMA 2.0 work.  Reasons are those given by Colin O'Brien.

2. ODMA Workflow Specification

 

Download the ODMA Workflow 1.0 Specification:

  • 1995-04-24 Specification: Microsoft Rich Text Format document, workflo3.rtf (31,801 bytes)
    Note: A 1995-11-27 edition of this specification is included in the ODMA Workflow 1.0 Software Development Kit.

 

3. ODMA Workflow Software

 

 

Download the ODMA Workflow 1.0 SDK:

  • Installation Instructions for Workflow 1.0 SDK: Windows Text File readmewf.txt 
    (1,412 bytes).  This information is not incorporated in the SDK.
  • Version 1.0.0 SDK: Windows Zip File odmawf.zip (153,783 bytes)
  1. The Zip archive contains a directory tree for all of the Workflow software, including DLLs for the Workflow connection managers.
  2. After downloading odmawf.zip, follow the instructions in readmewf.txt to preserve the directory structure.
  3. When you want to operate the ODMA 1.0 Workflow Managers, edit the .reg files as instructed and then operate them (by double-clicking).
  4. There are only binary modules (DLL-s) for the ODMA 1.0 Workflow Managers (16-bit and 32-bit).  The whereabouts of the source code is not known.
  5. Additional subdirectories carry libraries, implementations of a sample client and sample workflow engine, source code for the samples, and the ODMA Workflow 1.0 specification.
