You've reached the scaffolding for the ODMA /active/ODMref1.0 section. 

This is a folder for the ActiveODMA development of ODMref 1.0, an ODMA 1.0 reference implementation for ODMA Document Management System (DMS) Integrations.  There will be additional material as project formulation and development continues. 

The following folders are being set up for materials of the ODMref 1.0 project:

  • engage - for the engagement of observers and participants in the development of ODMref 1.0 and its TROSTing.
  • envision - for the envisioning stage of ODMref 1.0 development.  We apply the Microsoft Solutions Framework (version 3, specifically) process model to development of ODMref 1.0.
  • mitigate - for a set of risk mitigation experiments used to confirm that we have the development environment and tools needed to build an ODMA DMS integration DLL, and that we can do so to an useful level in the time available for the base implementation.
  • plan - for the overall planning discipline applied to development of ODMref 1.0.  (We will separate the technical design activities into a separate folder even though they would be included here under MSF v.3.)
  • status - as a single point for reporting the current status of the project, with links to historical and supporting materials.
  • tools - for identification of appropriate tools for use in accessing project materials, employing ODMref 1.0, testing and verifying ODMA installation (with or without ODMref 1.0), conducting TROST verifications and assessments, and in participating in the development and adaptation of ODMref 1.0.
  • TROSTing - for the application of TROST templates and procedures to the development, deployment, and operation of ODMref 1.0.  The ODMref 1.0 development was selected as an appropriate feasibility case for the TROST Pilot project.  Although ODMref 1.0 is usable without concern for the TROSTing aspects, those interested in working with ODMref 1.0 are encourage to participate in the validation of the TROST templates and how they are applied for ODMref 1.0.

Additional sections will be added as the project moves ahead.

  • Software that is developed as part of ODMref 1.0 delivery will be fully available in the ODMA Interoperability Exchange /dev area.
  • Planning materials and project documents are also included in the ODMA Interoperability Exchange Projects Catalog in the /projects area.
