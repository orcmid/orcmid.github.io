ActiveODMA is a software portfolio that extends usability of the AIIM Open Document Management API (ODMA) and middleware.  ActiveODMA provides configuration confirmation and troubleshooting tools.  The provisions for using the ODMA API will then be extended to support Java and .NET integration.  New ODMA fixtures, documentation, and sample code make ODMA Visual Basic- and script-friendly. 

ActiveODMA is in an early bootstrapping phase.  Initial software is being developed in the ODMdev ODMA Development Framework section of this site.  As the work on ODMJNI continues, source code will be moved to the SourceForge.net project.  All ActiveODMA software and related materials are provided under open-source licenses. 

Recent News

  • 2007-09-29: ODMJNI 0.59 Public Transition Candidate Defined
    Definition of the 0.59beta Public Transition Candidate is complete and work has begun on the second of five stages that will be integrated for the release.  This will provide the first look at a working version of those ODMJNI 1.0 API elements that are expected to be stable and preserved from release 0.60 and beyond.  The Definition provides links to additional materials that reflect the development and results of the four remaining stages.
       
  • 2007-09-16: ODMJNI 0.59 beta candidate being defined
    The 0.59 roll-up will identify interface elements that are deprecated for removal in the 0.60 public beta.  The 0.59 release will also include their replacements intended for use hereafter.  Both forms are available for use in transitioning existing code using 0.59.  This also applies to the confirmation and regression tests, which will be provided in both flavors.  Definition of 0.59 is now available for review and planning of adjustment to the changes.
      
  • 2007-07-28: ODMJNI Public Beta Preview Available
    The ODMJNI 1.0 Java-ODMA Integration Kit is now available for Public Beta Preview.  This initial ActiveODMA offering is being provided as a starter stage for development of the ODMA Development Framework first proposed at the end of 2005.  More information is in the ODMdev section of the ODMA Interoperability Exchange site.
       
  • 2007-07-28 (update) 2005-12-13: Tools for ODMA Development and Support (announcement)
    Public announcement of initial tools to be contributed to ActiveODMA over the last half of 2007 includes
    • ODMJNI 1.0 Java-ODMA Reference Integration
    • ODMdev Development Framework
    • ODMrun1.0 Test Runner
    • ODMref1.0 Reference Implementation
