1. Incident Summary (2001-03-06)

When ODMA queries are performed directly via an ODMA-aware application, rather than by the normal File | Open ... and DMS search interface, the ODMA-aware application may lock up.  It will be necessary to force the application to shut down to break the lock, which is caused by a loop in the ODMA Connection Manager.

Why is this considered critical?

The condition can easily occur in practice and will crash an ODMA-aware application with loss of any work in progress.  The defect will have users avoid using the ODMA Query capability when it is supported by the application and one or more DMS integrations.

2. Actions Toward Resolution

The following actions are proposed:

  1. Record the existence of of the problem and establish an incident report as notification and as documentation of the problem to be confirmed.  [2001-03-06: Completed]
  2. Inspect the ODMA 2.0 Connection Manager implementation and identify the conditions under which the software fails.
  3. Construct a repeatable demonstration of the problem, if needed. 
  4. Establish work-arounds for the problem until a production solution is available.
  5. Implement any needed correction in project P000902: ODMA Connection Manager Release 2.0.1.
  6. Evaluate the need for specific trouble-shooting procedures for situations in which ODMA Query is used.
