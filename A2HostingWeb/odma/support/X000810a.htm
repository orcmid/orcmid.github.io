I started the investigation and got through step 3a and stopped.  
I don't believe it's an ODMA issue, but rather a WP9 issue.  
Here's my findings:

WP9 Standalone (no ODMA present)
Windows at startup = 88% system resources
Run WP9 = 69% system resources
Open document off local drive = 69% system resources
Exit WP9 = 84% system resources

WP9 Networked (with either ODMA v1.5 or ODMA v2.0) 
Windows at startup = 89% system resources
Run WP9 = 70% system resources
Open document via PC Docs = 66% system resources
Close document/PC Docs = 68% system resources
Exit WP9 = 83% system resources

That's a new one!

We have not had any reports of memory leaks related to operation of the ODMA
Connection Manager, neither the ODMA 1.5 version that you have installed nor
the currently-supported ODMA 2.0 version.

I have recorded an incident report on this.  It is ODMA Incident Report
X000810.  You will find the report at

 http://www.infonuovo.com/odma/support/X000810.htm 

It proposes a number of investigation steps that I would like you to conduct
if you feel comfortable doing them.

Please let me know whether this investigation is something you can take on
to isolate the source of the resource hogging.

We have another critical incident involving WordPerfect for Windows version
9, but it appears to be unrelated to your experience on WIndows 95.  It is
http://www.infonuovo.com/odma/support/X000500.htm 

Thanks to the work by a number of WordPerfect users this summer, we are
compiling trouble-shooting information for use by people who are integrating
WordPerfect with document management systems via ODMA.  I welcome your
review of that material and any suggestions on how it can be made as useful
as possible.  I will send more pointers separately.
