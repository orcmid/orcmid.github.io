Summary:

The ODMA Sample DMS fails with Microsoft Word 2000.

With the Sample DMS installed as the default DMS (as set up by installing the ODMA 2.0 SDK), the failure occurs as follows:

  1. Start Word 2000.  Type in a few letters in the initial unnamed document.
  2. Press the Word 2000 Save icon.
  3. When the Word "Select File As Type" dialog occurs, accept the default settings and click OK.
  4. A junk file name is then displayed in the title of the document.
  5. In addition, Word may report a "File Permission Error" some of the time.

Inspection of the ODMA32 log file reveals that

  • Word performs an ODMNewDoc operation followed by ODMSaveAs.
  • Word does not call ODMOpenDoc and continue further as stated in ODMA documentation.

Analysis (2000-06-22):

It has been suggested that the test fixtures of the ODMA 2.0 SDK (version 2.0.0) can be made to fail.  This incident is remarkably similar to the behavior observed in user-critical situations using production DMS integrations. 

This opens up the possibility that we can use the test fixtures to duplicate the behavior that shows up in critical applications, since we have all of the code for the test fixtures and can revise them as necessary to provide extended testing and trouble-shooting capabilities.

Status:

Investigate and Confirm (2000-06-22):
1. Obtain detailed information on failures sufficient for reliable reproduction of the incident on a test configuration.
2. On confirming reproducibility of the incident, produce problem analysis.
New Direction (2000-08-22):
This has been confirmed as a symptomatic failure of Word 2000 integration when Unicode operation is not disabled for ODMA access.  More information is available in the updated incident report.
