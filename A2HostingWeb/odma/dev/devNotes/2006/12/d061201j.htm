1. Purpose and Scope

Following introduction of the 0.60beta public-beta release, the packaging (including documentation) and deployment of ODMJNI are improved so that it is easier to install and confirm as part of a production Java application.  There will be bug fixes and some hardening of the code.  The code tree is replicated on SourceForge and the project history and materials are made fully public and available. 

Further development of OdmNative100 and ODMA Core support are separated from the ODMJNI 1.0 development project.  While ODMJNI will continue to have dependencies on those elements, with need to synchronize from time to time, there will be separate development progressions and projects.  OdmNative100 and ODMA Core are intended for independent usage and other integrations as part of the progression to the ODMA 2.5 level.

2008-03-31-08:41 The 0.70 Deployment Release is Mostly Ceremony
The movement from 0.60 Public Beta to 0.70 Deployment (with improved packaging) is mostly comprised of a set of ceremonial collateral activities.  The provision of additional documentation, better tests, and possible samples can fall here using an ad hoc prioritization. 
   To support deployment of ODMJNI and also set the stage for the progression to ODMA 2.5, there will be work on ODMA Core, on active use of the ActiveODMA SourceForge project, and on supporting additions to the nfoWorks Toolcraft materials.  These are all for supporting developers and also providing additional means for participation of other developers in the cycle of learning and improvement that goes with further ODMA development.
