  • Latest version: The latest ODMJNI 1.0 Deployment material can be found via   <http://ODMA.info/dev/devNotes/2006/12/d061201b.htm>.
  • Previous version:  ODMJNI 1.0 0.25alpha API Regression <http://ODMA.info/dev/devNotes/2006/12/d061201d.htm>.   That distribution was a regression check of the 0.30alpha practical100 interfaces with the end-to-end functionality established in 0.20alpha.  It is obsoleted by the functionality of 0.30alpha and later releases.
  • Next version:   ODMJNI 1.0 0.40alpha Document-Update Features <http://ODMA.info/dev/devNotes/2006/12/d061201f.htm>.   This release extends end-to-end document-access to allow for modification of accessed documents.  The modifications are used to update the managed version, with any change to the properties of the managed version also made available for presentation to the Java application.  There is also correction of a defect that was carried over from 0.30alpha.
             
  • This version:   0.04 ODMJNI 1.0 0.30alpha Document-Access Features  <http://ODMA.info/dev/devNotes/2006/12/d061201e.htm>.   This release adds end-to-end document-access for read-only use of documents using the OdmConnection.chooseDocument() method.  The delivered OdmWorkingDocument interface has fully-functional OdmViewingDocument operations except for transferToNewDocument (not functional until 0.50beta along with acceptNewDocument).

DEFECT NOTICE: There is a defect in the reporting of dmsAvailable in the 0.30alpha OdmJniView implementation.  See the 0.01 ReleaseNotes for details.  This defect is repaired in the 0.40alpha release.  There is no plan for repairing the 0.30alpha version at this time.
