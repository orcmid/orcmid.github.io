The 0.58beta material was created to confirm initial packaging of the ODMJNI 1.0 runtime for use by Java applications and demonstrate all supported functionality.

This material is offered as a preview of the Java-ODMA Integration Kit prior to availability of the 0.60 Public Beta release.  Prerequisites for use of the material include

  • ability to determine the use of the package by inspection of the source code for the interfaces and the release notes for the 0.50beta through 0.58beta code drops.
    • The source code for the correct interface definitions is found at page d061001g: info.odma.practical100 0.56beta Evolutionary Development.
    • The runtime package of compiled classes and release notes for usage of 0.58beta.  The jar file can be used in Java tests and trial applications.  The runtime package is provided as download ODMJNI100-0.58beta.zip at page d061101n: info.odma.odmjni100 Runtime Packaging 0.58beta.
    • The full source-code and development tree for the ODMJNI 1.0 0.58beta version can be obtained via the instructions on page d061201: ODMJNI 1.0 Deployment 0.5x beta Function-Stabilization Updates.
        
  • understanding that there are breaking changes to the interfaces to be introduced in the 0.59beta transition candidate and the 0.60 public beta releases:
    • The info.odma.practical100.OdmFormat class will be replaced by an interface and a refactored implementation that does not offer public static methods except for wfAppId.
    • There will be slight adjustments between the info.odma.practical100.OdmViewingDocument and OdmWorkingDocument interfaces.  There should be no impact on programs that do not make direct use of OdmViewingDocument.
    • The conditions under which OdmError is thrown have been heavily restricted.  The exception can no longer be used to determine failure to provide a correctly-formatted parameter.  Parameters must be pre-filtered if there is any concern for possible format errors.
