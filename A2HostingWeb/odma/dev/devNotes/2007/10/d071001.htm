ODMJNI 1.0 Confirmation Testing

ODMJNI 1.0 does not start out with comprehensive automated tests.  Although confirmation tests are developed alongside the development of the ODMJNI 1.0 packages, these have been rather arbitrary, ad hoc tests.  There is no initial effort to provide comprehensive tests until stable definition and implementations of functions was accomplished. 

Few ODMJNI 1.0 tests are easily automated.  Any test involving end-to-end operation with an ODMA Document Management System integration that involves operator input is essentially manual.  This initial condition is a consequence of the limited provisions for confirmation testing (including regression checks) in the current ODMA 2.0 Connection Manager and its support. 

This folio consolidates material on confirmation testing of all aspects of ODMJNI 1.0.  The objective is to focus attention on identification of gaps in test coverage, test setups, and test scripting: manual or automated.

The objective is to achieve, some time in the 0.90 time frame, full automated testing along with the certification of the code and builds for the 1.00 release.
