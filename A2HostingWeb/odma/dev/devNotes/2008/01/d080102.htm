The ODMA Core is the foundation on which all ODMA implementation, maintenance and support is based.  It includes the artifacts by which software developers rely on ODMA in desktop software products.  ODMA Core materials were evolved in the progression through ODMA 1.0, 1.5, and 2.0.

Recent developments with ODMA, including the Java-ODMA Integration, have shown that the current ODMA Core is not in an appropriate form for direct use in development of professional-quality ODMA-based software.  It is time to reorganize and update all of the materials for more-appropriate and confident usage. 

Although the ODMA Core materials are to be reorganized and enriched, there will be complete conformance with all of the ODMA specifications and with current implementations using the original core materials.   Compatibility is to be preserved at all times.

The modernization of ODMA Core will begin with simple, careful steps:

  • Preservation of the Original Core Materials in the form originally produced

  • Identification and Packaging the ODMA Core in forms that are suitable as open-source distributions

This will be followed by additional modifications and supplementary artifacts for expanding the usability of the ODMA Core using current development tools.

Ultimately, the ODMA Core will be refined to the "ODMA 2.5" level that provides the bridge from ODMA32 to an anticipated ODMA64 "ODMA 3.0" refinement of ODMA support.  These conceptual versions are used as a way of supplementing ODMA 2.0 with improvements that provide a new foundation for ODMA64.

It is not assured that there will ever be an ODMA64.  This effort positions ODMA for that opportunity, whether or not it is found worth achieving.
