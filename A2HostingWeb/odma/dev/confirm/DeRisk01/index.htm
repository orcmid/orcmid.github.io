You've reached the scaffolding for the ODMA Interoperability Exchange /dev/confirm/DeRisk01 subproject. 

This is the software-project folder for the first risk mitigation step as part of confirming development arrangements for the ODMref 1.0 project.  The overall DeRisk exercises are designed to establish whether the necessary tools and development capability are in hand.

The time records for these activity are used to validate the estimation of effort for construction and testing of ODMref 1.0 and its companion test harness, ODMrun 1.0. 

We are adopting a test-first methodology for ODMref 1.0 development and all other ActiveODMA projects.  To support this, DeRisk01 is provided for simple confirmation of a compiler and library setup for making a simple console-mode application.  This is done in a way so that anyone else can shadow what we've done and verify the tests, duplicating all of the results we found.

Although there were more ambitions than that for DeRisk01, it was useful to freeze the work on DeRun01 at that point and move on to DeRisk02, the next step.  This is designed to get DeRun02 enough beyond DeRun01 to be able to load DLLs.

The materials are not organized in a presentable fashion, although everything has been collected for doing so.  This will happen later, around the time that ODMref engagement is initiated.

Here are some background notes on all of this:

  • First steps of the exercise occurred on 2005-03-03 as part of the TROST project.
  • The plan for the exercise was out of date.  The plan was simplified as part of an exception review concluded on 2005-03-22.
  • The setup for the exercise exposed a large number of unexpected steps needed to have minimal, stable dependency on Microsoft tools and the Windows Platform SDK that are available without charge.  Success with DeRisk01 fulfills on the conditions for Experiment #1.  Additional exercises will be integrated in the ODMref 1.0 development plan refresh after conclusion of Experiment #1.
  • All of the steps for gathering together the necessary resources and reproducing the exercise are documented and maintained here.
  • Recreating DeRisk01 is an useful starter exercise for shadowing the overall ODMref 1.0 and ActiveODMA development activities.
