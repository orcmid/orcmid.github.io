Objective: 
  1. Creation of DMS Integrations using Visual Basic and comparable languages for creation of Automation-friendly (or dual interface) COM servers.
  2. Permit Reliable Development and Confirmation by testing independent of ODMA Connection Manager and with comprehensive test fixtures.
  3. Introduce bridge mechanism for access to ActiveODMA DMS integration from standard ODMA32 Connection Managers

2. Constraints (2001-10-05)

The constraints are special conditions on satisfaction of the requirements:

  • ActiveODMA is delivered as a (self-registering?) .DLL file.
  • ActiveODMA operates as an automation-compliant, scripting-usable COM server that delivers functionality of the ODMA Connection Manager to desktop applications and to other components developed in Visual Basic and in scripting languages (VBScript, JavaScript, Python, etc.).
  • ActiveODMA will operate with any installed ODMA Connection Manager, for any version of ODMA
  • ActiveODMA delivers ODMA character strings and lists of character strings in a form consistent with the COM-server and ActiveX models for character manipulation and string storage.  Record sets are preferable to custom lists of strings.
