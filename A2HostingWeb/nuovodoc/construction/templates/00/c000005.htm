In addition to providing descriptive information and documentation, this page is also a template for standard, undecorated pages in sections of the site.  The template features depend on the use of Microsoft FrontPage for the development and maintenance of this material, and are only fully available when these web pages are accessed using Microsoft FrontPage.  The template elements can also be used by direct copy and paste.

1. Informal Template Usage - Summary

1.1 This page can be used as a boilerplate for a full page in the default style for standard-content (document engineering) pages. 

  • 1.1.1 This is used for all ordinary pages that are not part of the construction structure, a construction zone, or material of the ODMdev /construction section.  (Special pages, such as default pages for folders and professional-appearance pages have separate format and style rules.)
        

  • 1.1.2 This page is itself constructed using the applicable rules and inclusion even though it is technically construction material.  As a template, it deviates so that it can be used as a model for a standard page. 
        

  • 1.1.3 There are template components are used in Microsoft FrontPage for authoring-time inclusion and automatic updating of all pages that rely on an Include Pages.  This updating happens on the development site.  The pages that are have included material appear to be standard HTML with everything present, so there is no burden on the hosted-site server or the browser.  The included elements (especially those on this page) can also be incorporated in new pages using copy and paste.

1.2 To use this page as a template in Microsoft FrontPage, make a new version of this page via copy and paste within the Folder List or Web Site view.  Rename the file to reflect its intended use after customization.  There will be alterations of shortcuts to reflect the new location.  Some will need to be re-adjusted manually.   Make link adjustments while customizing the new page:

  • 1.2.1 Use File | Properties ... to change the title of the page
        

  • 1.2.2 Change the title in the middle table block at the top of the page.
        

  • 1.2.3 Change the "anchor block" in the right corner of the page top to provide the full context of the present page along with a version number and time stamp.  These are relative links to the appropriate folders in the hierarchy descending from the top level of the site to the location of this template.
        

  • 1.2.4 Keep the Hard Hat area image inclusion in the lower-left corner, delete it, or replace it as appropriate, depending on where the governing construction-zone is situated.  (See 2.2.)
        

  • 1.2.5 Keep the "You are ..." text in the middle block, supplement it, or replace it as appropriate.  This is where intellectual-property notices are also place.  Be careful to employ the appropriate notice and links for the page being constructed.
        

  • 1.2.6 Update the creation information in the lower right section of the page-bottom banner.  After the first check-in of the page under source control, use the check-in date as the new creation date.
        

  • 1.2.7 Delete this text section and the box at the top of the page, replacing it with your own page content.
        

  • 1.2.8 Introduce your own version history annotations, if desired.  Make sure the links in the history are as you want them to be.

1.3 The title block -- the table at page top -- can be copied by itself to produce a title block with the correct styles and the automatic inclusion of the correct left-section image for this site.

1.4 The construction block -- the table at page bottom -- can be copied by itself to produce a construction block with the correct styles,  the automatic inclusion of the left-side Hard Hat image and link, and the "you are navigating ..." information in the center section of the block.  The right-side material will be updated automatically by Microsoft Visual Source Safe, if used, but the "created ..." line must be updated manually.

1.5 The body style -- the background, font, and link colorings, must be manually edited into the <body stylesrc="c000005e.htm"> tag with recent versions of FrontPage (e.g., 2003 and later).  Although the use of a private tag for this purpose is deprecated and styles should  be applied instead, we are preserving this practice for use with older versions of Internet Explorer.   Strictly standards-compliant browsers, including Internet Explorer 8, will ignore this element, and some other device will need to be introduced instead.

1.6 There are other ways to insert templates to be customized using Visual Source Safe.  Also, there are better ways to make full-page templates.  The present information is provided as a provisional solution until there is a smoother general technique in place.

2. FrontPage "Include Page" Components

The use of include-page component features are not visible on the published versions of these web pages.  The effect of the components will be present, but the origination as a component is buried in a special HTML comment.   When the page is opened for editing in FrontPage, the component relationship is revealed and can be manipulated.

2.1 Title Image Link: c000005b

 

This component creates the Title block image by inclusion of page c000005b.htm.  The component provides the image associated with this site, a title expressing the name of the site, and a shortcut to the site itself at its official URL.  The image shown is the latest version before this particular page was transferred to a hosted site or mirror.

2.2 Construction Image Link: c000005c

 

This component creates the table-cell image by inclusion of page c000005c.htm.   The standard form of the image provides a "Construction Structure (Hard Hat Area)" caption and a link to the construction structure (index.htm) of the folder this template is contained in.

In general, it is necessary to copy rather than include, then update the picture properties to link to the proper construction structure.

 2.3 "You Are Here ..." Text: c000005d
You are navigating NuovoDoc

This component creates the table-cell text by inclusion of page c000005d.htm.  The component provides the text that we want to have present in the bottom strip of construction-structure pages of this site by default. 

 2.4 "You Are Here" License Text: c000005f
You are navigating NuovoDoc.
This work is licensed under a
Creative Commons Attribution 2.5 License.

This component creates the table-cell text by inclusion of page c000005f.htm.  The component provides the text that we want to have present in the bottom strip of construction-structure pages of this site that are placed under this particular license.

This inclusion can be used directly when we want to be very specific about licensing.  For the current site, the standard block provides the license anyhow.

3. FrontPage Style Inclusion 

The FrontPage Style inclusion used for this and other Construction Structure pages is in page c000005e.htm.  That page describes how the background and basic styles are set on that page and how they are used from other pages.  If manual editing is required, use <body stylesrc="c000005e.htm"> with the proper path to the style source.
