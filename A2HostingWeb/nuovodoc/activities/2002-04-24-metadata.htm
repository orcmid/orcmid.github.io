Notes: April 24, 2002

This is a notes place-holder for discussions at Hitachi on questions around managing metadata for DMA.

Here are the bullets from Chuck Fay's April 1999 slides that Hide has:

  • International Support
  • Dynamic discovery of repositories based on middleware registration mechanism

Power Features:

  • Cross-repository search
  • Property aliases for cross-repository search
  • Renditions
  • Dynamic discovery of metadata
  • Internet and WebDAV-ready

The diagram that is used for dynamic discovery of metadata has

Any object (class description:) -> Class description object (properties:) -> list of Property Description

Slide bullets: Dynamic Discovery of Metadata

  • Run-time discovery -- no static configuration required in application
  • Discovery includes
    • Display name & descriptive text
    • searchability & search operators
    • Selectability in search results
    • Data type
    • Value constraints

    My additions:

    • requirement and policy (system derived, hidden, required, read-only)
    • need to make read-only write-on-creation-only too.

Discussion

Suggested topics for discussion include:

  • economizing the customization of applications and DocBroker configurations in terms of metadata design and integration activity.
