Project Style

This is about the approach taken to conduct of the project, whether informal or high-structured and carefully-managed.  Identify the project style that is employed for a project.  More than one designation may apply to a single project.  Common terms for project styles include ad hoc, ... .

Ad hoc

Project Style.  This is the style of project carried out "playing it by ear," "by the seat of the pants," and a kind of randomly-progressive trial-and-error approach.  This is often how exploratory projects are carried out.  Elements of project management in ad hoc projects are notable for their absence.  There may be no plan, schedule, or structure for performance to plan.  There is neither measurement nor risk management.  Requirements are vague and there may be no well-understood completion condition.

It is very easy to fall into this style on solo projects.  The only difference made here is in explicitly accounting for its presence.

Bootstrap

Project Style.  This project-style variant is focused on putting something in place where there was nothing before, and using that as scaffolding for building higher.   The allusion is to lifting oneself by ones bootstraps.  The same idea arises in computer systems where the start-up process is a series of bootstrap functions where there is initially only enough operating system for loading operating system and establishing the configuration.  In peer-to-peer computing, there is also a bootstrap process around finding any peers to communicate and coordinate with.

The idea is that a bootstrap must be simple and provide a framework on which further expansion can occur.   Here, there is the example of having no project template for the project that produced the first project template for use on nfoWare.

Bootspiral

Project Style.  This project-style variant is similar to a bootstrap, combined with spiral development.
