{This page is a template for the initial bootstrap of a sequence of  folio pages for a progressively-versioned InfoNote.  Instructions of this form will provide details on the systematic customization of this page to become the first-version's placeholder.}

{NOTES:

  • This page can be omitted if P000006b is not being included at this time.  In that case, it is often wise to leave both the b slot unassigned and the c slot might be kept in reserve as well. 

  • This page by itself can be used as a placeholder for many other folio-internal pages beside the c-page.  It should not be used for an a-page because that's reserved for a Job Jar & Diary Page in new folios, whether there is one initially or not.

  • If P000006b is used, it provides a latest-version placeholder that remains until the first version is developed enough to present as a stable version.   When that happens the stable version page (e.g., this c page) is included as the content of the b-page using the Microsoft FrontPage IncludePage feature on the development site.  The idea is that once that little bit of customization occurs to the b-page the only future action is an occasional change of the page that is included as the body.}

{First Steps:
 
  1. The heading title of the page and the title banner, above, are customized in the same manner as the Diary & Job Jar page for the target folio.
  2. In the title block, above, the first two lines can be replaced by copying from an already-customized Diary & Job Jar page.  If not, perform that customization on the first non-job-jar page created internal to the folio.
  3. The remaining title lines should be replaced by an appropriate substitute.  The <head> title for the page should be updated (using File | Properties in FrontPage, for example).
  4. Similarly, the updated anchor-block path in the upper right can be replaced by copying the already-updated version from the InfoNote cover page or the Diary & Job Jar page.
  5. The anchor for this page itself must have the link text changed from i000006c and the link should be corrected to refer to the page being built.  This is what is known as a permalink: it refreshes the page showing the actual URL at which it is found in the browser address bar.  This breaks out of any framing, cloaking, and other devices that may disguise the URL of the host site that is presenting the page.
  6. Customize the text box, below, to provide links that are readable from material which has been printed or otherwise detached.  The commentary should be restated to reflect the location and nature of the page being customized using this template.  If there is no need to preserve context at all, delete the text box instead.
  7. Provide customization at other places, below, where the {Customize} marker appears.  Delete markers (and other {...} text groups) when they have been acted upon.
  8. The list of placeholder content materials can be retained if P000006c.txt and P000006c.zip are also brought
over and customized to be minimal placeholders.  Otherwise, delete the list or make a different arrangement.  Remember to adjust the file names, links, and related information.
  9. Don't forget to adjust the revision history and the construction-history block as specified at the bottom of this page.}
