{This page is a template for the initial bootstrap of a sequence of  folio pages for a progressively-versioned InfoNote.  Instructions of this form will provide details on the systematic customization of this page to become the latest-material placeholder until there is initial material.}

{NOTES:

  • This page can be omitted if there is no content other than the "cover" and a possible job jar for getting started. 

  • This b-page is typically the first content page or else a virtual "latest" page that is maintained by automatic inclusion of a copy of the body of the current actual "latest" content (using a Microsoft FrontPage feature for accomplishing that). 

  • This template assumes that the virtual-page approach is being used.  It also assumes that there is no actual latest content available just yet, so it is a stand-in until the first content is ready to see the light of day. 

  • If there is no need for a virtual "latest" in this particular InfoNote, this page can simply be the placeholder for the first content itself.  If there is any concern that there may need to be a virtual "latest" page in the future, the b-slot can instead be left unused until such time (so long as there are no early links to it).} 

{First Steps:
 
  1. The heading title of the page and the title banner, above, are customized in the same manner as the Diary & Job Jar page for the target folio.
  2. In the title block, above, the first two lines can be replaced by copying from an already-customized Diary & Job Jar page.  If not, perform that a-page customization on this page and further pages can then copy from here.
  3. The remaining title lines should be replaced by an appropriate substitute.  The <head> title for the page should be updated (using File | Properties in FrontPage, for example).
  4. Similarly, the updated anchor-block path in the upper right can be replaced by copying the already-updated version from the InfoNote cover page or the Diary & Job Jar page.
  5. The anchor for this page itself must have the link text changed from i000006b and the link should be corrected to refer to the page being built.  This is what is known as a permalink: it refreshes the page showing the actual URL at which it is found in the browser address bar.  This breaks out of any framing, cloaking, and other devices that may disguise the URL of the host site that is presenting the page.
  6. Customize the text box, below, to provide links that are readable from material which has been printed or otherwise detached.  The commentary should be restated to reflect the location and nature of the page being customized using this template.  If there is no need to preserve context at all, delete the text box instead.
  7. Provide customization at other places, below, where the {Customize} marker appears.  Delete markers (and other {...} text groups) when they have been acted upon.
  8. Don't forget to adjust the revision history and the construction-history block as specified at the bottom of this page.}

Placeholder Status {Customize}

This page is a placeholder for material which is yet to be incorporated.  When it is available, this page will be updated to provide further details as well as links for supporting materials. 

The use of provisional pages like this is akin to the automatic creation of ready-to-edit pages for new Wiki Words in a wiki.  This provisional page template is designed for manual creation, but the purpose is similar: To provide some target as the destination of an already-created link, even though the desired content is not yet available.  It allows visitors to know that there is an intention to provide material here and the link that reached this place is not broken.  It also provides some indication of the nature of content that can be expected to be developed over time.
