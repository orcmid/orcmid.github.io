URI <http://nfoworks.org/ns/example#> is the Namespace Identifier used for nfoWorks Namespace and Identifier examples.  Resolution of the URI to a web page is a policy for nfoWorks URI-based identifiers. 

1. The Namespace

1.1 The {http://nfoworks.org/ns/example#} namespace is used solely to illustrate the use of Namespace-identifying URIs established for nfoWorks purposes. 

2. myTerm

2.1 Local Name Usage

2.1.1 This namespace has but one defined local name: "myTerm".  The use of the term

{http://nfoworks.org/ns/example#} myTerm

is defined at the URI

<http://nfoworks.org/ns/example#myTerm>

under the policy for nfoWorks URI-based Namespaces and Identifiers.

2.1.2 This local name has no significance apart from illustrative definition of local names for an nfoWorks URI-based Namespace. 

2.1.3 It is also the case that this definition for local name myTerm has nothing to do with the same name in the identifier URI  <http://nfoworks.org/ns/example/myTerm>.

2.2 URI Identifier Usage

2.2.1 The URI <http://nfoworks.org/ns/example#myTerm> is hereby defined to be an identifier for {http://nfoworks.org/ns/example#} myTerm.  This allows the term to be identified via a URI, in a CURIE,  and in RDF triples.  This allows usage beyond those provided by use of XML Namespaces.  For this URI, the significance as an identifier is the same as the significance of the namespace-qualified local name (2.1.1).  They are defined together.

2.2.2 The URI usage as an identifier is in addition to the URI usage as the location of the definition for name {http://nfoworks.org/ns/example#} myTerm.  Usage as an identifier is specific to this precise URI and no other URI, even if it resolves to the same definition location. 

2.2.3 There is no requirement that any URI for locating the definition also be understood as an identifier of the namespace-qualified term.  The definition will determine whether there is an identifying URI and what it is.
