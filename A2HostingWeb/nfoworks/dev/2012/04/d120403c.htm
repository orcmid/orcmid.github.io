 		nfoWorks-issued identifiers beginning with http://nfoworks.org/ns/odf/1.2/ are for use with OpenDocument Format (ODF) 1.2 documents.  These are managed under the ns/odf/ Namespace and Identifier structure.

1. Identifier-Segment Usage

1.1 The next path segments are for namespace identifiers of form
 
{http://nfoworks.org/ns/odf/1.2/feature#}

1.2 The feature segment identifies a category of ODF 1.2 provisions for which  nfoWorks-defined features are established.  The detailed definition for each namespace is found at the corresponding web-site location <http://nfoworks.org/ns/odf/1.2/feature#>.

2. nfoWorks ODF 1.2 Extension Features

nfoWorks-defined ODF 1.2 extensions are sometimes declared for provisional before an ODF 1.3 Committee Draft defines the feature and reserves identifiers for it.    Reserved identifiers cannot be produced until ever allowed in and OASIS Standard ODF specification.   Provisional use is with the  nfoWorks-defined identifiers. 

Some extension features are intentionally provided as independent extensions and there is no effort to reserve corresponding OASIS-issued identifiers.

Note: It is permissible to accept but not produce an OASIS-reserved identifier.  If the reserved feature is adopted unchanged, the reserved identifier will become available for use and will begin to appear.  If the feature is not adopted, the provisional use can continue as an extension or an implementation-dependent/-defined feature and the reserved identifier, although anticipated, will never appear in documents.

There are presently no OASIS-reserved counterparts of the following nfoWorks identifiers.

2.1 PCT

Namespace {http://nfoworks.org/ns/odf/1.2/pct#} provides namespace features for the nfoWorks PCT ODF Protected Change-Tracking Extensions.

2.2 Protection

Namespace {http://nfoworks.org/ns/odf/1.2/protection#} provides implementation-defined features that extend the ODF 1.2 protection-key attributes and the corresponding digest algorithms.

2.3 RCT

Namespace {http://nfoworks.org/ns/odf/1.2/rct#} provides extension-profile features of the RCT Repaired Change-Tracking ODF 1.2 Extension Profile.

2.4 Security

Namespace {http://nfoworks.org/ns/odf/1.2/security#} provides extensions for the ODF 1.2 encryption methodology, including derivation of cryptographic keys from passwords.
