The nfoWorks URI <http://nfoworks.org/ns/example/myTerm> is an example of an nfoWorks standalone URI-based identifier:

  • It is resolvable exactly as spelled.
  • It is not defined to be a namespace.  It is simply a single identifier.
  • Were it a namespace that would be specified here.
  • It is convenient to think of simple identifiers that do not contain fragment parts (#...) as being their own namespaces, since they are unique.  There is no such usage in this particular case.

There is no relationship defined between this URI and any other form ending in myTerm.  There is no connection with the unique identifier <http://nfoWorks.org/ns/example#> nor the term <http://nfoWorks.org/ns/example#myTerm>.

These are incidental features of this particular identifier that apply generally for URI-based nfoWorks identifiers:

  • The resolution is to a folder on the nfoWorks web site. 
  • The behavior of the web server in providing a default page is relied upon to supply the content.
  • There are many URIs that resolve to the same resource.  However, only the one that is defined to be the identifier is the identifier.  Use of other URIs may have some value (i.e., perhaps in an RDF context), but none of the alternatives are the identifier.

Note 1: There is no intended purpose for identifier <http://nfoworks.org/ns/example/myTerm> beyond serving as an example.  It is also introduced for contrast with the interpretation of the different identifier, <http://nfoworks.org/ns/example#myTerm>.  nfoWorks identifiers are recognized as distinct in the same manner as XML Namespace names.  This is the case despite existence of other forms that resolve to the same web-site location.

Note 2: There is no nfoWorks Namespace {http://nfoworks.org/ns/example/} with local name myTerm.  That is the case despite the fact that the particular URL is resolvable to the definition page for Namespace {http://nfoWorks.org/ns/example#}
