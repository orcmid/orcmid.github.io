You've reached the scaffolding for the nfoWare /dev/ab section. 

The /dev/ab folder holds the source code and other software elements for implementation of an addbib implementation.  This particular implementation uses an identified and versioned format for addbib materials, so they can be customized and extended for specialized applications.  This development is also an experiment in bootspiraling of a well-defined format starting with provisional forms that are maintained manually.

This is not a particularly interesting starting point for development of nfoWare utilities except that it does allow ideas about always-documented formats and simple data entry using them to be applied early.  It happened that the creator of this site is in need of an addbib utility that works more generally than the few that are around.

There is not much content here at this point.  This scaffolding serves as a place to hold the existing schemas until additional structure is provided.  It's a bootspiraling chicken-and-egg situation.

  • ab-about.txt is a little file that provides some background on what is being developed here.  It is not in the formal format that is used for full-up developments.  That comes later.
  • ab-0.02.bib is a text file that serves as the version 0.02 schema of the ab addbib format.  This at the URI that is carried in copies of .bib files that are based on this schema, as reflected in their #! prologue lines.  It is here ahead of its documentation and it is being used in the creation of that documentation as well as other trial-use applications.

This index.htm page is part of the engineering, construction management, maintenance, and infrastructure of the nfoWare.com site.  You are welcome to explore this and other construction-information pages.   You'll need your hard hat and safety shoes.

If you'd rather find content on nfoWare development, go directly to the dev section.   If that doesn't work, come back later.  The section is blocked off whenever the construction crew is blasting in the area.
