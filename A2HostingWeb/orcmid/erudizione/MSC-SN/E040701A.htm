used in DQ 2004-07-04 There is an MD5 cracking service.  It was slash-dotted yesterday and offers to crack MD5 hashes from a small space using a time-space tradeoff and machines with lots of memory.  They are currently cracking about 150 hashes every 24 hours.  Salted md5's are not cracked, and I will need to figure out what that is and how it works.  The references are to the Project RainbowCrack, which has many tools, and cracking solutions for LM (LAN Manager), MD5, and SHA1, with other contributions for further schemes, links to tutorials, and a key paper by Philippe Oechslin that basically searches two CD-ROMs to crack 99.9% of all alphanumeric password hashes for MS Windows in 13.6 seconds.  Document this and figure out how to try it.  I downloaded the paper.
  2004-07-04 In a console session, I created a text file, orcmid.pwd, carrying a six-character password in a pattern that is similar to a password that I have and use.  I then used the console md5sum application to create an orcmid.md5 file that confirms the password file.  The md5sum --help application provides what is needed.  Document this and explain where I got the application, where others can get the application. [note: Making the file in an editor or by echo causes a CR-LF on the end.]
  2004-07-03 There is a Passware program that can be booted from CD-ROM and it will reset the Administrator password on Windows XP.  This basically allows a machine to be attacked, a special account set up, or other activities.  There is also a utility for recovering the password underneath an *-filled password field on a web form or open dialog box.
  2004-07-03 In Windows 95 there was a vulnerability around leaving the Password Cache lying around, Knowledge Base Article 140557.  This depended on finding *.pwl files.
  2004-07-03 A vulnerability in the Microsoft Password system was described on 2004-06-07 in a Beta News Article.  However, the flaw was originally reported on 2003-07-24.  This requires access to the one-way encrypted (or hashed) password.
  2004-07-03 Here is the DQ2.1" Find one example of password protocols in Microsoft product and discuss its security or insecurity with your colleagues. If possible, install one of the available cracking tools to see whether you could crack this system and share your experience with your colleague."
  2004-07-03 Ways users don't manage passwords well [Anderson2001: 3.3.3]: 
 1. Using the same password for more than one purpose
  2004-07-03 Ways in which password clues become available [Anderson2001: 3.4]:
 1. Audit trails showing bad login attempts may reveal passwords when users type them out of order with user Ids.
| 2. Ways to capture encrypted passwords and crack them off-line.
  3. Man-in-the-middle for capturing passwords
  2004-07-03 It is important to consider the threats we are dealing with [Anderson2001: 3.5]
 1. Are we protecting people from each other?
 2. Can the people be trained and disciplined?
 3. Can accounts be frozen after a certain number of bad guesses? (permits DOS attack)
 4. Will attackers be after a particular account?
 5. Will break-in to any account (of a class) be sufficient?
 6. Are we concerned about snooping by malicious software, false terminals, or network eavesdropping?
 7. Is there a vulnerability to off-line password guessing?
