IdmaVersionSeries2

This interface provides extensions of the IdmaVersionSeries interface to facilitate versioning of complex objects such as compound-structured documents.  This is draft version 0.01 of 1999-11-23 of the proposal for this interface.

All methods of IdmaVersionSeries are supported by this interface.  In addition, the following interface is supported.

IdmaVersionSeries::SetExtendedCheckOutNext

Support for this method is optional.

Syntax
DmaRC IdmaVersionSeries2::SetExtendedCheckOutNext (
Dmapv pReservation)

Parameters

Name Mode Description
pReservation input Pointer to an interface of a new, unsaved reservation object

Description

Performs all of the operations of IdmaVersionSeries::SetCheckOutNext except the working Versionable object is constructed differently. 

For SetExtendedCheckoutNext, the working Versionable object is an extended clone of the current Versionable object in the version series.  The general effect is to also make clones of those relationships for which the checked-out Versionable object is at the tail, with the same objects at the heads of those relationships.  For example, if a referential container were versionable, the initial clone would share the same referentially-contained content.  A checked-out CSRoot will share the original CSComponents and a checked-out Versionable object with virtual elements will have the same virtual elements.

When the extended clone is created, the versionable object and all dependent persistable subordinate objects are copied recursively. All system-derived properties (such as dmaProp_This, dmaProp_OIID, and dmaProp_InstanceId) that are specific to a given (dependent) object are given appropriate new values for the clone. 

Independently-persistent objects that are directly reachable by navigation from the original Versionable object (including any of its dependently-persistent subordinates) may have counterparts -- projections -- in the extended clone according to the following rules:

  1. An independently-persistent object is projectable into the extended clone if and only if
    1. It is not Versionable;
    2. It has a dmaProp_Tail property the value of which is the original Versionable object;
    3. It is reachable by navigation through an object-valued property, or EnumOfObject element of the original Versionable object (including any of its dependently-persistent subordinates); and
    4. The independently-persistent objects that can be directly reached by navigation from the independently-persistent object through elements other than dmaProp_Tail have no restriction against being reached by an additional independently-persistent object in the same way(s).
  2. Exactly one projection is cloned for each projectable object, regardless of the number of ways that it will be reachable by navigation from the extended clone.
  3. When the projection is created,
    1. the original projectable object and all dependent peristable subordinate objects are copied recursively;
    2. All system-derived properties that are specific to a given (dependent) object are given appropriate new values for the projection;
    3. The dmaProp_Tail property of the projection is set to be the extended clone and any related properties (e.g., ones with InstanceId values that apply to the tail object) are defined accordingly;
    4. All other navigation elements to independently-persistent objects from the projectable object are copied into the projection -- the same objects become reachable by navigation in the same ways from the projection.
  4. When the an object-valued property of the original Versionable object has a projectable object as its value, the extended clone has the projection as the value of that property.   Otherwise the value of the property in the clone is determined in the same way as for SetCheckOutNext.
  5. When an EnumOfObject property of the original Versionable object has projectable objects as its elements, the corresponding property of the extended clone will deliver the projections of those projectable objects.  Otherwise the value of the EnumOfObject property in the extended clone is determined in the same way as for SetCheckOutNext.

When the original Versionable object has no projectable objects reachable by navigation, these rules lead to the same result as the IdmaVersionSeries::SetCheckOutNext operation.  The treatment of ListOfObject properties with values that are not dependently persistent is the same as for the IdmaVersionSeries::SetCheckOutNext operation.

The reservation created when the Extended Check Out is executed can be resolved with a check-in, or can be revoked. Revoking a reservation created as a consequence of SetExtendedCheckOutNext does not delete the working Versionable object.

Return Values

Name Description
DMARC_BAD_OBJECT An inappropriate object was passed as parameter to this function. For example, the object does not support a required interface or is of an inappropriate class.
DMARC_CONFLICTING_OPERATION The call on this primary intentional method logically conflicts with a prior call on a different primary intentional method.
DMARC_NO_MEMORY (E_OUTOFMEMORY) Insufficient memory to complete the operation.
DMARC_NOT_SUPPORTED This method is not supported in the context of this session or object.
DMARC_OK (S_OK) Success.
DMARC_RESERVATION_PENDING The connection is already holding a request for a reservation. There cannot be more than one at a time.

Deferred Return Values

Name Description
DMARC_NO_CURRENT_VERSION The version series has no current version.
DMARC_OBJECT_LOCKED The lock(s) on the persistent object prohibit the attempted operation.
DMARC_OBJECT_MODIFIED The object has been modified since it was retrieved.
DMARC_RESERVATION_EXISTS The version series is already holding a reservation.
DMARC_RESERVATION_NOT_ALLOWED The condition of the version series does not permit a reservation, though no reservation is currently present.
