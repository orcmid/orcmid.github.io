DMA - WebDAV Convergence Analysis

These tables of notes compile areas of intersection, disagreement, and absence of features to be examined for DMA-WebDAV Convergence.   They are based on early meetings and E-mail discussions and are neither comprehensive nor validated through careful scrutiny and agreement on concepts.

This information is posted for early review and to elicit additional thinking by those involved in exploring DMA-WebDAV convergence: WebDMA.

I am personally optimistic that there is a feasible DMA - WebDAV convergence.  And there are many details to wrestle with.  This compilation is meant to get us thinking, not scare us off.

-- Dennis E. Hamilton

Compound-Structured Documents different none

Arbitrary Property Sheet and Name Space differences XML expression of property information

object interface expression of property information without serial form

DMA namespace mapping to XML namespaces requires work.  Might fit the DMA alias principle in a different form.
Unconstrained Property Set differences WebDAV sites can allow each object to have unique set of properties DMA object properties strongly tied to class descriptions and DocSpace schema, which is indelible for duration of a connection DMA extension to support import-preservation-export of arbitrary bundle and projection onto DMA property schemata of the object is worth exploring.

defined transfer (Copy/Move) procedures

no wholesale interchange or import/export model

Popular Properties different introducing Dublin Core

Document Models different none model for managed documents and for renditions as content of managed documents Look at DOM for WebDAV, projection of DMA onto WebDAV
Class, Property, Type Model differences none?

implementation may have restictions on objects, but WebDAV schemel does not model it

 Strongly Typed with Explicit Class Model and Inheritance Rules DMA view inversion around rendition = resource may resolve these
Metamodel and Schema differences none? All objects carry descriptions and there is an explicit metamodel for classes and properties of every repository  
Referential Container differences none

Advanced Collections corresponding to Referential Containment or Mixed Containment?

 Container can employ both referential and direct containment.

Containables (including containers) can be contained referentially (many times) and directly (at most once).  Containment-relationship objects provide edge differentiation.

  
Relationship Model differences none? Explicit well-defined relationship model with relationships carried in content-less "resources"  
Authorization differences Access Control proposal none Is WebDAV providing resource-level or finer individual authorization?   Can it be used for that for DMA?  DMA can get value from this.
User Model differences HTTP principals? none
