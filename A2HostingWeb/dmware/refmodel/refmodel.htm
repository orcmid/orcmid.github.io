Here are two recent presentations.

They are in PowerPoint 97 format in WinZIP files.

  • AIIM 99 Reference Model Presentation

    This is from the DMA Forum at AIIM 99.  It contains most of the slides of the latest DM Reference Model discussions.  It is expanded slightly to illustrate two ways that CORBA services can be utilized in DMA distributed integration.

    I would make the following additions to this model:

    • I would remove Workflow Management from the IT infrastructure.  Instead, I would put Messaging and Notification in the infrastructure.  This was done in the new working presentation as of 1999-06-11.  I also added storage services to the infrastructure.

      I would make Workflow Management a peer service and present it in much the same way as Document Management.  It would allow illustration of the different integration models that are supported for Workflow, just as for Document Management.   This was done on 1996-06-12 in the new working presentation.

    • I would make Document Management (content), Workflow Management (process), and Knowledge Management (analysis and synthesis) into peer level functions that are used cooperatively.   This is best illustrated with a pie diagram atop an infrastructure plane.   This allows the different contacts between the three to be expressed in terms of the DM Reference Model's integration view.
    • I would differentiate the DM Reference Model as an integration model, in contrast with the 3-tiered model used for distributed enterprise applications, Microsoft DNA, and OMG CORBA Services.
  • Global DM Perspective: Japan, May 1999

    This is a more-recent presentation, using much of the Reference Model material, that gave me more insight into how to present this.  The changes that I propose, above, are based on my experience using the materials in the May 1999, Japan presentations.  This is the latest version, used in Osaka on May 28, 1999.
