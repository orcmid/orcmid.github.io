How does this relate to ISO 11179, standard metadata described in a metadata repository?

Dan Schneider provides:

What the repository is holding is a description of the metadata elements, and there is a schema for that.  It appears that ISO 11179 addresses this.

ISO TC32 experiments are working to bring repositories into alignment with ISO 11179.  JDC1 SC32, meeting in Seol, Korea, going through a big revision and update to 11179 around practice and reconciliation.

The XML repositories want to be aligned with the 11179 specification.  The coordination of XML-11179 reconciliation is through OASIS.

So 11179 provides a standard for metadata description and having repositories reflect that specification.

11179 should be addressed here so that holdings can be corresponded and holdings-information can be interchanged.  The more consistency in the description of the information holdings

Regarding expertise in AIIM, not clear there is any.  There is work in the engineering community and the EPA has a fairly advanced and sophisticated system.  The FAA has been working internationally within the aviation community to align US aviation safety data with that of other countries.  Department of transportation doing so with ITS, the Intelligent Transportation System.  DOT has subcontracted with the IEEE to build their metadata repository subcontractor.

Two other communities: Health care community, with big projects in HIRS, for military health and HICFA of HHS in finance and administration.  Australia is a leader in this area and US people are working with them.  The other is the education community.  There has been a project to describe educational objects in standard metadata so they can be discovered and understood and shared more easily.  E.g., if a college builds a course module to be available to others, they can describe it in a standard way.

Neale: Library Community also heavily involved, NISO focused strongly on bibliographic metadata, and has a specification going into trial use now.
